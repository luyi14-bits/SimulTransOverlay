## 1. 产品定位

SimulTransOverlay 是一款 Windows 桌面实时同声传译工具。它捕获系统正在播放的全部音频（视频/直播/会议），通过本地 AI 语音识别转写为文字，再通过本地翻译引擎译为目标语言，最终在一个始终置顶的半透明叠加层中显示译文。

**核心卖点**：零外部依赖、全程本地处理、隐私安全、双击即用。

## 2. 系统架构

### 2.1 四层异步管线

```
┌─────────────┐    audio_queue     ┌─────────────┐    segment_queue
│  音频捕获线程  │ ──(60 max)────→  │  VAD 处理线程 │ ──(20 max)────→
│ (WASAPI回调)  │                   │ (Silero VAD) │
└─────────────┘                    └─────────────┘
                                        │
                                        ↓
┌─────────────┐    text_queue      ┌─────────────┐
│  翻译调度线程  │ ←──(10 max)─────  │  ASR 推理线程  │
│ (Ollama/Builtin)│                 │ (faster-whisper)│
└─────────────┘                    └─────────────┘
        │
        ↓ (QTimer.singleShot 主线程安全回调)
┌─────────────┐
│  UI 主线程   │  ← PyQt6 半透明叠加层更新
└─────────────┘
```

### 2.2 线程隔离策略

| 线程 | 职责 | 解耦方式 | OOM 防护 |
|------|------|---------|---------|
| 音频捕获线程 | WASAPI 回调 → resample → push 到 `audio_queue` | 纯回调，无阻塞 | Queue maxsize=60，满时丢弃旧帧 |
| VAD 工作线程 | 从 `audio_queue` 读取 → VAD 分句 → speech segment 到 `segment_queue` | 独立 daemon 线程 | Queue maxsize=20 |
| ASR 工作线程 | 从 `segment_queue` 读取 → faster-whisper 推理 → text 到 `text_queue` | 独立 daemon 线程 | Queue maxsize=10 |
| 翻译工作线程 | 从 `text_queue` 读取 → 翻译 → `QTimer.singleShot` → UI 更新 | 跨线程信号 | Queue maxsize=10 |
| 内存监控线程 | 每 5 分钟检查 RSS，触发 `gc.collect()` | 独立 daemon 线程 | MemoryMonitor |
| 模型生命周期 | 空闲 5 分钟自动卸载 ASR 模型，语音活动时热加载 | ModelLifecycleManager | 显存释放 |

### 2.3 背压策略

当下游处理速度慢于上游生产速度时，队列满后自动丢弃最旧的数据帧，防止内存无限增长。所有队列都有明确的 maxsize 上限：

```
audio_queue:    60 frames  (~2s 音频)
segment_queue:  20 segments (~10s 语音)
text_queue:     10 segments
translate_queue:10 segments
```

## 4. AI 引擎

### 4.1 ASR 双引擎架构

```
create_asr_engine(engine_name)
    │
    ├── "faster-whisper" → WhisperASREngine
    │       ├── model: base/small/medium
    │       ├── device: auto (CUDA → CPU)
    │       └── compute_type: int8/float16
    │
    └── "sensevoice" → SenseVoiceASREngine
            ├── 模型: iic/SenseVoiceSmall
            └── VAD+标点 联合推理
```

**faster-whisper**（主引擎）：
- 基于 CTranslate2 的 Whisper 加速实现
- GPU 自动检测（CUDA）→ 降级 CPU INT8
- 支持 99 种语言，识别精度 BLEU > 30（中→英）

**SenseVoice**（备选引擎）：
- 阿里达摩院 FunASR 框架
- 中/英/日/韩 四语种优化
- 需安装 FunASR 依赖包

### 4.2 翻译引擎

**BuiltinTranslator**（默认）：
- 基于 ctranslate2 + OPUS-MT 模型
- 4 个预置语言对：ja→zh, ja→en, en→zh, zh→en
- 模型首次运行时自动下载（~50MB/对），后续完全离线
- 推理延迟 <200ms（CPU INT8）

**Ollama 备选**：
- 适用需要更高翻译质量的场景
- 用户需自行安装 Ollama + qwen2.5:3b

### 4.3 模型生命周期管理

```
状态图:
[未加载] → mark_activity() → [加载中] → load_model() → [已加载]
                                                          │
                                                   空闲 5 分钟
                                                          │
                                                          ↓
[已卸载] ←──── unload_model() ←──────────── [空闲超时]
```

- 空闲 5 分钟 ⇒ 自动卸载 ASR 模型释放显存
- 检测到语音活动 ⇒ 自动热加载
- 每 30 秒检查一次空闲状态

### 4.4 模型管理器

- 自动下载：ModelScope / HuggingFace 双源
- 缓存路径：`%APPDATA%/SimulTransOverlay/models/`
- 完整性校验：SHA256
- 断点续传：支持
