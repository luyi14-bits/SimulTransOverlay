## 2. 纵深防御体系

### 2.1 第一层：音频隔离

- WASAPI 环回捕获仅在用户态进程内运行
- 音频数据生命周期：WASAPI 回调缓冲区 → 重采样 → VAD 推理 → 释放
- 不写入文件系统，不通过网络传输

### 2.2 第二层：队列解耦

- 4 级 Queue 隔离：音频数据/语音段/转写文本/翻译结果
- 每级 Queue 有独立 maxsize 上限 + 背压丢弃
- 崩溃或异常不会在队列间传播

### 2.3 第三层：内存水位监控

```python
class MemoryMonitor:
    def check(self) -> bool:
        process = psutil.Process()
        total = process.memory_percent()
        if total >= self.threshold:  # 默认 80%
            gc.collect()            # 强制 GC
            return True
        return False
```

- 每 5 分钟触发一次检查
- 超过 80% 阈值 ⇒ 强制 `gc.collect()`
- 冷却期 60 秒避免频繁触发

### 2.4 第四层：模型生命周期管理

- 空闲 5 分钟 ⇒ ASR 模型卸载释放显存
- 语音活动检测 ⇒ 自动热加载
- 避免 GPU 显存长时间占用

### 2.5 第五层：去匿名化打包

- `build_release.ps1` Stage 4 自动检测 `.pdb`、`.env`、`.db` 等敏感文件
- `.gitignore` 覆盖：`*.pdb` `*.toc` `*.pkg` `*.pyz` `dist/` `build/`
- PyInstaller `--windowed` 模式隐藏控制台窗口
- `--exclude-module test` `--exclude-module notebook`

## 3. 安全模块详解

### 3.1 API Key 与凭据管理

| 组件 | 凭据类型 | 存储方式 | 安全等级 |
|------|---------|---------|---------|
| Ollama | 无 | 本地服务 localhost:11434 | ✅ |
| DeepSeek (可选) | API Key | 环境变量建议 | 🟡 config.yaml 默认空 |
| HuggingFace | 无 | 公开模型下载 | ✅ |
| GitHub | Token | gh CLI keyring | ✅ |

### 3.2 配置脱敏

`config.yaml` 中 `translation.deepseek.api_key` 默认空字符串。用户如使用 DeepSeek 应通过环境变量注入而非直接写入 config 文件。

### 3.3 结构化日志

- JSON 格式：时间戳 / 级别 / 模块 / 消息 / 关键指标
- 自动轮转：单文件 max 10MB，保留最近 10 个
- 存储路径：`%APPDATA%/SimulTransOverlay/logs/`
- 诊断导出：一键打包脱敏日志 + 配置摘要 → `diagnostics.zip`

### 3.4 异常处理规范

- 所有 except 块必须包含 `logger.error()` 记录
- 零容忍 `except: pass`（全代码库审计通过）
- 全局 `sys.excepthook` + `threading.excepthook` 捕获未捕获异常
- 崩溃时自动 dump 线程栈 + 最近 100 条日志
