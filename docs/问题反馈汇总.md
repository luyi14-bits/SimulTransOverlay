## 3. 已解决问题详解

### 3.1 #9 — sounddevice 惰性导入遗漏 (🟢 已修复)

**问题**：`audio_capture.py` 中 `import sounddevice as sd` 放在 `start()` 方法内部，PyInstaller 静态分析扫描不到，打包后运行时崩溃。

**根因**：PyInstaller 的静态分析无法追踪函数内部的 import 语句。

**修复**：将 `import sounddevice` 提到模块顶部（`audio_capture.py` 第 8 行），同时在 PyInstaller 命令中添加 `--hidden-import sounddevice`。

**验证**：`dist/SimulTransOverlay/_internal/sounddevice` 存在，运行 `import sounddevice` 无报错。

### 3.2 #10 — psutil 惰性导入遗漏 (🟢 已修复)

**问题**：`pipeline.py` 中 `import psutil` 在 `MemoryMonitor.check()` 方法内部，PyInstaller 未打包。

**修复**：将 `import psutil` 提到模块顶部（`pipeline.py` 第 23 行），添加 `--hidden-import psutil`。

### 3.3 #11 — onedir 分发需复制 _internal/ 目录 (🟢 已修复)

**问题**：Release 附件只上传了单个 exe，缺少 `_internal/` 依赖目录，用户下载后无法运行（`python312.dll` 找不到）。

**修复**：将完整 `SimulTransOverlay/` 目录（含 `_internal/`）打包为 `SimulTransOverlay-v1.0.zip`（326MB），替换 Release 附件。

**验证**：`gh release view` 显示 zip 附件，解压后 exe + _internal 共存。

### 3.4 #12 — SteelSeries Sonar 设备不可选 (🟢 已修复)

**问题**：`list_loopback_devices()` 返回的是 WASAPI 输入设备（麦克风），而环回捕获应使用输出设备（`max_output_channels > 0`）。

**根因**：sounddevice 中，WASAPI 环回捕获通过 `InputStream` 打开**输出**设备来捕获其播放的音频。

**修复**：`list_loopback_devices()` 改为返回 `max_output_channels > 0` 的设备，支持按名称搜索和自动评分选择（Gaming > Media > Stream > ...）。

**设备优先级**：评分制（关键字匹配度 + 通道数奖励），自动选择 SteelSeries Sonar - Gaming。

### 3.5 #13 — ASR 语言写死日语 (🟢 已修复)

**问题**：`config.yaml` 中 `language: "ja"`，中文/英文视频时 Whisper 按日语解码，输出乱码。

**修复**：`config.yaml` 改为 `language: "auto"`，`asr_whisper.py` 中 `language=None` 触发 Whisper 自动语言检测。

## 4. 未解决问题详解

### 4.1 #1 — DeepSeek API Key 明文配置 (🟡)

**描述**：`config.yaml` 中 `translation.deepseek.api_key` 默认为空字符串，但若用户填写则明文存储在磁盘。

**影响**：其他进程可读取 config.yaml 获取 API Key。

**建议修复**：改用环境变量 `SIMULTRANS_DEEPSEEK_KEY`，config.yaml 中仅保留占位符。

**优先级**：低（DeepSeek 为非默认引擎，默认使用内嵌翻译）。

### 4.2 #2 — 魔数未命名 (🟡)

**描述**：以下几处魔法数字应提取为命名常量：

| 位置 | 魔数 | 含义 | 建议常量名 |
|------|------|------|-----------|
| audio_capture.py:14 | `1536` | 32ms @ 48kHz 的采样点数 | `BLOCKSIZE_48KHZ_32MS = 1536` |
| config_loader.py | `48000`, `16000` | 采样率 | `SAMPLE_RATE_INPUT = 48000`, `SAMPLE_RATE_ASR = 16000` |
| main.py | `512` | 32ms @ 16kHz 的采样点数 | `CHUNK_SIZE_16KHZ_32MS = 512` |
| main.py | `60`, `20`, `10` | 队列最大长度 | `QUEUE_MAXSIZE_AUDIO = 60` |

**影响**：代码可维护性降低，修改参数需多处同步。

### 4.3 #3 — 背压测试线程竞争 (🟡)

**描述**：`tests/test_pipeline.py::test_worker_backpressure` 测试中，停止工作线程与 drain 队列存在竞争条件，导致测试挂起。

**影响**：无法运行完整 80 测试套件，需 `-k "not test_worker_backpressure"` 跳过。

**建议修复**：将 `PipelineWorker.stop(drain=True)` 的 drain 逻辑改为在 worker 线程内完成，而不是在 calling thread 中直接消费队列。

## 5. 架构演进关键决策

| 决策 | 旧方案 | 新方案 | 原因 |
|------|--------|--------|------|
| 翻译引擎 | Ollama（外部服务） | BuiltinTranslator（内嵌 ctranslate2） | 零外部依赖 |
| 许可证 | MIT | AGPL v3 | 完全开源 |
| 打包策略 | --onefile（3GB 失败） | --onedir（52MB + 模型按需下载） | 构建效率 + 分发体积 |
| 设备检测 | device=None（默认设备） | find_best_device()（自动评分） | 适配 SteelSeries Sonar |
| ASR 语言 | language="ja" | language="auto" | 多语言支持 |
| 退出机制 | 仅托盘菜单 | 叠加层 ✕ 按钮 + Ctrl+Shift+Q | 可用性 |
